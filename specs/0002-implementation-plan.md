# 0002 实现计划

## 1. 文档信息

- 文档编号：`0002`
- 文档名称：视频字幕提取与翻译软件实现计划
- 上游规格：`specs/0001-spec.md`
- 目标：将 `0001` 中已确定的产品与技术约束拆解为可执行的开发计划

## 2. 实现目标

首版实现以下闭环能力：

- 基于 `Rust + Tauri 2` 的桌面应用框架
- 导入单个/多个视频文件并形成任务列表
- 选择输出目录，默认输出到视频同目录
- Whisper 模型下载、管理、模型目录落盘
- 音频抽取、转写、生成 `srt`
- 可关闭翻译，或使用 OpenAI 兼容 LLM 进行翻译
- 三种字幕生成模式
- 任务开始、软暂停、删除、清空
- GPU/CPU 检测、Whisper GPU 开关、推荐模型档位
- 术语表、翻译批次、翻译失败回退
- 严格便携模式的数据/配置/密钥存储

## 3. 总体实施策略

### 3.1 开发原则

- 先打通单任务完整链路，再扩展为批量与并发
- 先保证 `关闭翻译 + LLM 翻译` 两条正式路径稳定，再考虑实验性翻译引擎
- 先实现任务执行可靠性，再补性能优化
- 先统一数据模型与任务快照机制，再做 UI 联动
- 先交付最小可用桌面版，再补高级体验

### 3.2 范围边界

本计划只覆盖 `0001` 已确认的首版正式范围。

不纳入本计划主线：

- Google Web 免费模式的稳定交付
- 字幕编辑器
- 多字幕格式
- 自动恢复未完成队列
- 多 Whisper 后端并行支持

## 4. 建议架构

### 4.1 技术栈

- 桌面容器：`Tauri 2`
- 核心语言：`Rust`
- 前端：建议 `React + TypeScript`
- 异步运行时：`tokio`
- 序列化：`serde`
- 配置格式：`json`
- 任务状态持久化：首版可用 `json`，后续可迁移 `sqlite`
- 音频抽取：`ffmpeg`
- HTTP：`reqwest`

### 4.2 分层架构

建议分为以下层次：

1. `UI 层`
   - 主界面
   - 配置页
   - 模型管理页/区域
   - 任务详情/错误信息展示
2. `应用服务层`
   - 任务编排
   - 配置快照生成
   - 输出路径决策
   - 翻译管线调度
3. `领域层`
   - 任务模型
   - 配置模型
   - 字幕模型
   - 术语表规则
4. `基础设施层`
   - 文件系统
   - 加密存储
   - `ffmpeg`
   - Whisper 适配器
   - LLM 适配器
   - 硬件检测

## 6. 数据模型与状态设计

### 6.1 配置模型

至少包含：

- `translator.engine`
- `llm.base_url`
- `llm.model`
- `whisper.model`
- `whisper.use_gpu`
- `whisper.recognition_lang`
- `translate.source_lang`
- `translate.target_lang`
- `translate.style`
- `subtitle.mode`
- `subtitle.output_dir_mode`
- `subtitle.custom_output_dir`
- `runtime.max_parallel_tasks`
- `runtime.task_auto_retry_max`

### 6.2 任务模型

至少包含：

- `id`
- `video_path`
- `file_name`
- `status`
- `progress`
- `phase`
- `translator_engine`
- `config_snapshot_id`
- `config_snapshot_summary`
- `output_paths`
- `error_message`
- `created_at`
- `updated_at`

### 6.3 状态流转

建议任务状态流转如下：

`待开始 -> 排队中 -> 提取音频中 -> 识别中 -> 翻译中 -> 导出中 -> 已完成`

异常分支：

- 任意阶段 -> `暂停请求中` -> `已暂停`
- 任意阶段 -> `失败`
- `失败` -> `开始` -> 重新执行
- `已暂停` -> `开始` -> 继续执行

### 6.4 配置快照策略

任务在点击开始时生成快照，并写入任务记录。

快照必须至少覆盖：

- 翻译引擎
- 字幕模式
- Whisper 模型
- 是否使用 GPU
- 识别语言
- 翻译源语言/目标语言
- 输出目录
- 输出覆盖规则
- 术语表版本
- 翻译风格
- 最大重试次数

## 10. 风险与应对

### 10.1 Whisper 后端集成风险

风险：

- GPU 支持与推理库兼容性不稳定

应对：

- 首版仅选一个实际可落地的 Whisper 后端
- CPU 路径必须可用
- GPU 不可用时自动降级或明确提示

### 10.2 LLM 接口兼容风险

风险：

- 各家 OpenAI 兼容实现细节不同

应对：

- 统一抽象请求结构
- 保持适配层隔离
- 连接测试返回清晰错误

### 10.3 批量翻译错位风险

风险：

- LLM 返回段落不一致导致字幕错位

应对：

- 严格编号校验
- 失败重试
- 自动缩批
- 最终逐条回退

### 10.4 便携模式安全风险

风险：

- 密钥虽然加密，但仍和软件放在同目录

应对：

- 明确 UI 提示边界
- 不记录明文 Key
- 加密文件和配置文件分离

### 10.5 Windows 目录权限风险

风险：

- 用户将软件放到不可写目录

应对：

- 启动时检测软件目录写权限
- 无权限时阻止关键流程并明确提示
